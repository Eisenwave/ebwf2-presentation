Run-Length Encoding

  • First used in 1950s for transmitting TV signals
  • Input: string of characters
  • Output: string of pairs (character, run-length)
  • Strategy: collect long runs of characters into one marker

Example

\[abaaaaaabbbbb \rightarrow (a,1)(b,1)(a,6)(b,5)\]

Example with Character-Pair RLE

\[\begin{align} ababababbaaaab &\rightarrow (ab,4)(ba,1)(aa,1)(ab,1) \end{align} \]

Lempel-Ziv Compression (LZ77)

  • Developed by Abraham Lempel, Jacob Ziv in 1977
  • Input: string of characters
  • Output: string of pairs (backwards-distance, length)
  • Strategy: convert repeated words to backwards-references

Example Input

\[\text{THE BROWN ELEPHANT IS THE BEST ELEPHANT}\]

Example Output

\[\text{THE BROWN ELEPHANT IS }(22,3)\text{ BEST }(21,8)\]

Huffman Coding

  • Developed by David A. Huffman, 1951
  • Input: set of character and their probabilities: $(e, 12.02\%), (t,9.10\%), \ldots$
  • Output: binary code for every character: $(e, 1), (t, 01), \ldots$
  • Strategy: construct binary tree of characters

JPEG / Discrete Cosine Transform

  • DCT first developed by Nasir Ahmend in 1972
  • Input: 8x8 block of pixels
  • Output: combination of 64 discrete cosine patterns
  • Strategy: convert 8x8 blocks of pixels using DCT

Sources:
https://en.wikipedia.org/wiki/JPEG#/media/File:Dctjpeg.png

Video Compression

  • Early example: H.261 developed by the Video Coding Experts Group, 1988
  • Input: sequence of video frames
  • Output: sequence of compressed frames

Types of Frames

  • I-Frame: Intra-Coded Frame
  • P-Frame: Predicted Frame
  • B-Frame: Bid. Predicted Frame
